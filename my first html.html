¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet se refiere a la infraestructura, la red de redes que permite la comunicación entre dispositivos haciendo posible la comunicación y transferencia de datos.
La web es la parte del Internet que permite la presentación y acceso a información y recursos multimedia, esta es accesada gracias a los navegadores web

¿Cuáles son las partes de un URL?

Las partes de una URL son:

  1. protocolo (https://) --- indica el tipo información que se estará consultando
  2. subdominio (www) --- refiere a World Wide Web y es el subdominio más común, de http y https por ejemplo
  3. dominio (ejemplo) --- es el nombre del sitio web
  4. extensión o TLD (.com) --- es la parte que que clasifica sitios web por tipo o geografía, influyendo en la confianza y el SEO
  5. la ruta o subcarpeta (/pagina) --- identifica el apartado y/o archivo al que se pretende llegar dentrod e un mismo sitio web
siendo en este caso la URL completa: https://www.ejemplo.com/pagina
También existen elementos "ocultos" o que se omiten la mayoria de veces, como los puertos, que en caso de http utiliza el "80"

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita la representación de un recurso específico. Se utiliza para consultar información sin alterar el estado del servidor.
  • HEAD: Funciona igual que GET, pero sin devolver el cuerpo de la respuesta. Ideal para verificar disponibilidad, caché o tamaño de un recurso sin descargar datos pesados.
  • POST: Envía datos al servidor para crear un nuevo recurso o realizar acciones que cambian el estado.
  • PUT: Reemplaza o actualiza un recurso existente por completo con la nueva representación enviada.
  • PATCH: Aplica modificaciones parciales a un recurso, actualizando solo los campos enviados sin necesidad de re-enviar todo el recurso.
  • DELETE: Elimina un recurso específico del servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

En el caso de enviar datos sensibles como usuarios y contraseñas de un sistema web, el método que se debe de utilizar es POST, debido a que este método no permite que los datos a manejar viajen de forma insegura y completamente pública por la red, dandole un cifrado y hacviendo que este tipo de datos no sean filtrados a personas indebidas.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Para accesar a una página web atraves de HTTP se utiliza el método GET, el que nos permite obtener, consultar y posteriormente mostrar información solicitada. Por lo tanto es importante considerar que parámetros se envian y de que forma al soliictar acceso a una página, debido a que estos seran pasados por la red abiertamente.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Los códigos de HTTP determinan el estado de una página tras una acción (generalmente), estas son identificadas con números. El código 200 rerpesenta un funcionamiento correcto de la página, por lo que ningún error se ha dado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 representa que una solicitud fue rechazada debido a la inexistencia del recurso al que se intenta accesar, eso puede ocurrir al escribir erroneamente la dirección de un sitio o dando click a un link mal diseñado.
En este último caso es responsabilidad del diseñador no presentar direcciónes confusas o equívocas para el acceso de usuraios, sin embargo en cualquier otro caso es comúnmente un error del usuario.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Un error 500 indica que el servidor web encontró una condición inesperada que le impidió procesar la solicitud, eso es generalmente por envio de parametros no comprehensibles para el sistema, archivos corruptos, o solicitud de script

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

AL pasar de versiones (como de HMTL 4 a 5) algunos atributos se marcar para su "jubilación", siendo que dejaran de operar como se ha estado haciendo, esto se puede deber a una reestructuración por la actualización, una optimicación o una falta de uso en la nueva versión; sin emabrgo, etso no significa que el atributo dejara de funcionar, simplemente avisa que ya no es actual y posteriormente se inhabilitar. Algunos atributos despreciados de HMTL 4 al 5 son:
  • center: se utilizaba para centrar contenido, actalmente se utiliza las funciones de css como -- text-align: center; --
  • font: servía para cambiar el color, tipografia y diseño en general de un texto, actualmente se utiliza las funciones de css como -- color, font-family, font-size. --
  • u: utilizado para subrayar texto, actualmente se utilizan funciones de css como -- text-decoration: underline;. --

¿Qué componentes de estructura y estilo tiene una tabla?

  1. Componentes de Estructura (HTML)
    • table: El contenedor principal que envuelve toda la estructura
    • caption: El título o descripción breve de la tabla (opcional, pero excelente para accesibilidad).
    • thead: Agrupa el contenido del encabezado.
    • tbody: Agrupa el cuerpo principal de los datos.
    • tfoot: Agrupa el pie de la tabla (útil para sumatorias o notas).
    • (Table Row): Define una fila
    • (Table Header): Define una celda de encabezado (el texto suele salir en negrita y centrado por defecto)
    • (Table Data): Define una celda de datos estándar.
  2. Componentes de Estilo (CSS)
    • border-collapse: Es la más importante. Decide si los bordes de las celdas están separados o se unen en una sola línea fija (collapse).
    • border: Define el grosor, estilo y color de los bordes.
    • padding: El espacio interno entre el texto y el borde de la celda.
    • text-align: Alineación horizontal (left, center, right).

Considero que el uso de este comando es útil para formular una estilización general de una página web sin enecsidad de indagar en la estructuración de ID's, clases, etc. mostrando exactamente lo que uno desea, sin embargo no me parece correcto para la presentación final de un projecto pues demuestra en muchos casos una programación desestructurada.

Es importante considerar las imagenes o color de fondo debdio a que es una presencia estética constante en nuestras páginas web que afecta a todo el contenido, el cual es más diverso en color, forma y de más; siendo que un fondo elegido sin cuidado puede afectar a la visualización de nuetsra página.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Yo recomiendo el uso de porcentaje (%) pues el uso de pixeles y pt puede afectar a la visualización entre dispositivos, siendo que estos dos últimos son medidas "generales" y el "%" depende de el tamaño de la página donde se este presentando el HTML.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

El uso de css minimizados eficienta el procesamiento de la misma al reducir espacios inecesarios que le toman tiempo a la maquina, siendo que más recursos son invertidos en procesar lo importante por lo que menos tiempo le toma en mostrar el resultado.

Material Design (Framework: Bulma)


Material Design es el sistema de diseño de código abierto creado por Google en 2014. Material Design se rige por tres principios que lo hacen sentir natural y coherente:
  1. Material en un espacio mallado: Los elementos ocupan un lugar en el espacio (eje Z). Tienen superficie, bordes y sombras. Si un botón está "encima" de otro, proyectará una sombra más profunda.
  2. Llamativo, gráfico e intencional: Utiliza tipografías grandes, mucho espacio en blanco y colores vibrantes para guiar la atención del usuario hacia lo más importante (como el famoso botón flotante de "redactar" en Gmail).
  3. El movimiento con significado: Las animaciones no son solo adornos; sirven para explicar de dónde viene un objeto. Si tocas una foto y se expande, el movimiento te confirma que esa foto es el origen de la nueva pantalla


¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Semejanzas

  1. Sintaxis basada en C: Ambos heredan la estructura de llaves { }, puntos y coma ; y la forma de escribir bucles (for, while) y condicionales (if).
  2. Orientación a Objetos: Ambos pueden trabajar con objetos y clases, aunque lo hacen de formas internamente diferentes.
  3. Bibliotecas y Ecosistema: Los dos cuentan con comunidades gigantescas y una gran cantidad de librerías para resolver cualquier problema.


Diferencias

Característica Java JavaScript
Tipo de Lenguaje Compilado (se transforma en bytecode antes de correr). Interpretado (el navegador lee el código línea a línea).
Tipado Fuerte y Estático: Debes definir si una variable es int, String, etc. Débil y Dinámico: Una variable puede ser un número y luego un texto.
Plataforma Se ejecuta en la JVM (Java Virtual Machine). Se ejecuta principalmente en Navegadores y Node.js.
Uso Principal Aplicaciones empresariales, Android, Backend masivo. Desarrollo web (Frontend), apps móviles híbridas, servidores.
Verbosidad Muy descriptivo (necesitas muchas líneas para algo simple). Mucho más conciso y flexible.

¿Cómo se declara una variable con alcance local dentro de una función?

Las variables creadas con el operador "let" especifica una variable (de tipo dinámico) que existe solo bajo su rango de operación, por ejemplo dentro de una función, un ciclo, etc.
Al generarse con el operador "var" la variable no deja de existir, siendo que pueda representar una mala implementación al dejar variables inutilizadas tras un pequeño espacio.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Al declarar variables globales dentro de funciones corremos el riesgo de mandar a llamar o utilizar dichas variables fuera de la función y generar problemas. Es común que variabels usadas en funciones no se usen fuera de ellos al ser muy locales, como por ejemplo los iteradores.

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es importante esta forma de validación de datos puesto que permite la retroalimentación de la forma sobre alguna respuesta antes de que esta sea enviada, esto fomenta una eficiencia y repidez al rellenar formas y libera de craga al servidor con respeustas incorrectas. Sin embargo, no se deberia considerar unicamente este método para la validación de respuestas en formas, puesto que esta puede ser evitada o "saltada" por un usuario, al ser el código ejecutado en el navegador.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Al desactivar el JS dentro del navegador puedes desactivar las acciones/ordenes que envia para verificación de datos en una forma.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Este refiere a que es una buena práctica complementaria y mas de "ayuda" al usuario sobre de seguridad, puesto que permite la verificación ent iempo real de los datos para confirmar su correcta inscerción, sin embargo no debe de ser el único método de validación de datos en una forma puesto que como vimos, se pueden saltar.
